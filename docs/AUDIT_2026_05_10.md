## CRITICAL defects (must fix before shipping disclosures)

### D1. Tier 1 disclosure templates render `UndefinedError`

**State:** ❌ Broken. Workers cannot trigger via dashboard; bearer-protected
`POST /disclosures/render` would crash for CA / NY / FL / GA.

**Cause:** `compliance/disclosure.py:_build_context()` emits only metadata
fields (`bill_number`, `statute_citation`, `verified_date`, etc.). The
Tier 1 templates require ~20 *computed* disclosure variables that this
function never derives:

| Template var (CA SB 1235 sample) | Source needed |
|---|---|
| `funding_provided` | money math — gross advance |
| `recipient_funds` / `recipient_funds_lt_funding` | net delivered |
| `pays_third_party_payoffs` / `third_party_payoff_note` | renewal payoff |
| `apr` | IRR via `scoring.apr` |
| `payment_channel` / `avg_monthly_income` | merchant statement context |
| `finance_charge` / `finance_charge_can_increase` | money math |
| `estimated_total_payment` / `estimated_payment_amount` / `estimated_payment_freq` | scoring outputs |
| `payment_terms_text` | pre-rendered branch by structure |
| `estimated_term` / `estimated_term_explanation` | scoring outputs |
| `has_monthly_cost_insert` / `estimated_monthly_cost` / `monthly_cost_derivation` | money math + § 942 |
| `prepayment_finance_charge_text` / `prepayment_additional_fee_text` | contract terms |
| `irregular_payments_note` | optional |

NY / FL / GA templates each have their own similar variable set.

**Existing test `test_template_renders_via_disclosure_router_tier1_path`
acknowledges the gap:**

> "the router currently passes a smaller context, so we patch to assert the
> template is the one selected (full render is covered by the unit test
> above)."

The unit test renders by hand-passing a complete context — it does **not**
exercise `_build_context()`. So the gap is invisible to `make check`.

**Partial fix applied this audit:** added `financer_name` setting
(`Settings.aegis_financer_name`, default `"Commera Capital"`) + injected
into `_build_context`. This removes one of ~20 undefined errors; the rest
remain.

**To complete:** implement a `_build_tier1_disclosure_context(...)`
helper that derives each variable per the template's required-variable
docstring, gated by an integration test that calls `render_disclosure()`
end-to-end for CA/NY/FL/GA and asserts no `UndefinedError`.

**Risk if shipped as-is:** the bearer-protected API would 500. Workers
can't trigger from the dashboard today, but a future feature that wires
disclosures into the merchant detail page would break. CLAUDE.md is
explicit: *"the disclosure HTML for each state MUST match the template
prescribed by that state's regulator"*.

### D2. `/healthz` is gated by Cloudflare Access

**State:** ⚠ Operational impact only — not a security or correctness defect.

**Symptom:** External monitoring (Cloudflare Healthchecks, Uptime Robot,
etc.) cannot ping `/healthz` because the Access rule covers every path.

**Options:**
1. Add a Cloudflare Access bypass rule for `path = /healthz` (recommended;
   the endpoint is metadata-only — version + DB ping).
2. Run a CF Worker that signs a service token and probes `/healthz`.
3. Rely on systemd `Restart=always` and accept zero external monitoring.

**Recommendation:** (1) — bypass rule for `/healthz` and `/ui/healthz` only.

## Operational notes for workers

- **Sign-in:** Click `https://aegis.commerafunding.com` → email PIN to
  the allow-listed address → Access sets a 24h session cookie → dashboard
  loads. Same flow regardless of which page the worker bookmarks.
- **Adding a worker:** Edit the Access policy via the Cloudflare dashboard
  or run `scripts/cf_provision_tunnel.sh worker_email@domain.com`
  (idempotent, will update the allowlist).
- **Upload flow:** `/ui/intake` is the canonical worker entry point —
  create merchant + drop 1–4 PDFs in one request. Parsing happens in the
  background (arq worker on the box); merchant detail page shows progress.
- **Findings export:** Every merchant detail page has a "Download CSV"
  link → flattened audit-trail-ready spreadsheet. EIN is masked out by
  policy.
